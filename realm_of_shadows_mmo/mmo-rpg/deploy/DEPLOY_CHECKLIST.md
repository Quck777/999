## 🔒 БЕЗОПАСНОСТЬ

### Аутентификация
- [x] Пароли хешируются через `password_hash()` (bcrypt, cost=12)
- [x] Подготовленные запросы PDO (защита от SQL-инъекций)
- [x] CSRF-токены во всех формах и AJAX-запросах
- [x] Rate limiting на авторизацию (5 попыток / 15 минут)
- [x] Сессии с `httponly`, `secure`, `SameSite=Strict`
- [x] Генерация токенов через `random_bytes()` (CSPRNG)

### Защита данных
- [x] XSS-эскейпинг: `htmlspecialchars()` на всех выходных данных
- [x] Валидация всех входных данных на сервере
- [x] Фильтрация email через `filter_var(FILTER_VALIDATE_EMAIL)`
- [x] Запрет HTML-тегов в чат-сообщениях
- [x] Минимальная/максимальная длина строковых полей

### Конфигурация сервера
- [x] .env файл вне webroot
- [x] Запрет доступа к скрытым файлам (.env, .git)
- [x] Запрет доступа к /storage/, /logs/, /config/, /app/
- [x] Запрет доступа к .sql, .log, .json файлам
- [x] X-Frame-Options: SAMEORIGIN
- [x] X-Content-Type-Options: nosniff
- [x] Content-Security-Policy заголовки
- [x] Gzip сжатие для статики

### Боевая система
- [x] Все расчёты урона на сервере
- [x] Серверная валидация каждого хода
- [x] Клиент отправляет только action — сервер возвращает результат
- [x] Таймауты боя (60 секунд на ход)
- [x] Прерывание боя при AFK/disconnect

## 🚀 ДЕПЛОЙ

### Системные требования
- **PHP**: 8.2+ (pdo, pdo_mysql, mbstring, json, openssl, session)
- **MySQL**: 8.0+ или MariaDB 10.6+
- **Nginx**: 1.18+ (или Apache 2.4+ с mod_rewrite)
- **PHP-FPM**: 8.2
- **ОЗУ**: мин. 512 MB, рекоменд. 1-2 GB
- **Диск**: мин. 5 GB SSD

### Шаги деплоя

```bash
# 1. Клонирование репозитория
git clone <repo-url> /var/www/mmo-rpg
cd /var/www/mmo-rpg

# 2. Настройка окружения
cp .env.example .env
nano .env  # Заполнить DB_*, APP_*

# 3. Установка зависимостей (Composer — если используется)
composer install --no-dev --optimize-autoloader

# 4. Создание БД и импорт схемы
mysql -u root -p < database/schema.sql

# 5. Настройка прав
chmod 600 .env
chmod -R 775 storage/ logs/
chown -R www-data:www-data .

# 6. Настройка Nginx
cp deploy/nginx.conf /etc/nginx/sites-available/mmo-rpg
ln -s /etc/nginx/sites-available/mmo-rpg /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx

# 7. Перезапуск PHP-FPM
systemctl restart php8.2-fpm

# 8. Запуск скрипта деплоя
bash deploy/deploy.sh
```

### Структура Production

├── logs/            ← Запрещён доступ из web

### Мониторинг и обслуживание
```bash
# Проверка статуса
systemctl status nginx php8.2-fpm mysql

# Логи
tail -f /var/www/mmo-rpg/logs/app.log
tail -f /var/log/nginx/mmo-rpg-error.log

# Бэкап БД (cron — каждый день в 03:00)
0 3 * * * mysqldump -u backup_user -p'pass' mmo_rpg | gzip > /backups/mmo-rpg-$(date +\%Y\%m\%d).sql.gz

# Очистка старых бэкапов (оставить 30 дней)
find /backups -name "mmo-rpg-*.sql.gz" -mtime +30 -delete

# Очистка кэша rate limiting (каждый час)
0 * * * * find /var/www/mmo-rpg/storage/cache/rate_limit -name "*.json" -mmin +60 -delete
```
